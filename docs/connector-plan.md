Описание принципа работы коннектора:

Микросервис на FastAPI c тремя режимами работы:
1. Webhook listener: api endpoint, который принимает запросы от siem. В качестве теста - Wazuh, который имеет отдельный модуль для интеграции внешних сервисов через endpoint /webhook. В данном сценарии SIEM будет сам отправлять данные в json формате

2. GET запросы: http get запросы к API индексера siem, в котором лежат уже с коррелированные события . Сервис делает запрос и получает набор событий по дате или id алерта/события

3. POST запросы: отправка уже обработанных данных обратно в  сием для визуализации

После получения алертов из системы они нормализуются в этом же коннекторе в формат ECS
После уже все нормализованные события уходят в брокер сообщений
